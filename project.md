## 데이터 ↔ 보고서 연결

`server.js`의 `FIELDS` 배열이 **영문 변수명 ↔ 한글 엑셀 헤더**를 매핑한다.
설문 폼(`index.html`)의 `name` 속성, 서버 저장 키, 보고서 변수명이 모두 동일 코드로 연결되어
응답이 쌓이면 곧바로 대시보드 ② 양식으로 집계 가능하다.

| 변수명 | 엑셀 헤더 |
|---|---|
| timestamp | 제출시간 |
| name / age_group / gender / visit_freq | 이름 / 연령대 / 성별 / 구매빈도 |
| score_taste/aroma/texture/appearance/temp | 맛 / 향 / 식감 / 외관 / 온도제공 |

# 🔄 2차 확장 — QR 설문 + 구글폼 종합정리 (5인 8라운드 토론)

목표: **두 갈래 시스템**을 완성한다.
- **시스템 1 (자체 설문):** QR → 각자 폰에서 작성 → 내가 한눈에 종합
- **시스템 2 (구글폼 정리):** 구글폼 스프레드시트 파일을 넣으면 자동 종합 정리

## 전문가 5인
| # | 전문가 | 관점 |
|---|---|---|
| A | 시스템 아키텍트 | 폰 접속·호스팅·QR 인프라 |
| B | 데이터 분석가 | 집계 로직·구글폼 컬럼 매핑·통계 |
| C | UI/UX 디자이너 | "한눈에" 가독성·대시보드 |
| D | F&B 현장 운영자 | 시식 현장 QR 운영·응답률 |
| E | 자동화 엔지니어 | 비개발자 편집·파일 업로드 변환 |

## 8라운드 토론 요약 (비판 → 합의)

**R1. localhost는 폰에서 안 열린다 (A vs D)**
D: "현장에서 QR 찍었는데 안 열려요." A: localhost는 그 PC 전용. 합의 → 폰은 **같은 WiFi의 PC 내부 IP**(예: `http://192.168.0.10:3000`)로 접속. server.js가 켜질 때 내부 IP를 자동 출력하도록 반영함.

**R2. WiFi가 안 되는 외부 행사라면? (A vs D)**
D: "외부 팝업스토어엔 사내망이 없어요." A: 3가지 옵션 — ① 사내 시식회=LAN IP(무료·즉시), ② 외부=무료 클라우드 배포(Render/Railway에 server.js 올림), ③ 임시=ngrok 터널. 합의 → **기본은 LAN, 외부는 클라우드 배포**를 권장으로 문서화.

**R3. 결과는 어디서 보나 — 실시간? (C vs B)**
B: "엑셀 열어 매번 계산은 비효율." C: 전용 대시보드 필요. 합의 → `/api/results` + **`results.html` 관리자 대시보드**. 15초 자동 갱신으로 사실상 실시간. CEO용 출시 신호등을 최상단 배치.

**R4. 구글폼은 컬럼이 제각각이다 (B vs E)**
E: "폼마다 질문이 달라 고정 매핑 불가." B: 그래서 **고정 스키마를 강요하지 말고, 어떤 파일이든 자동 분류**하자 — 숫자형/선택형/주관식을 자동 판별. 합의 → `import.html`이 컬럼 유형을 자동 감지하고, '만족/추천(NPS)' 키워드는 핵심 KPI로 자동 승격.

**R5. 두 경로를 한 대시보드로 합칠까? (A vs C)**
A: "통합하면 코드 복잡." C: 사용자 입장에선 입력 소스가 다르니 화면도 분리가 직관적. 합의 → **화면 2개로 분리하되 디자인·통계 로직은 동일**(Claude 디자인 + 평균/NPS/편차/키워드 공통). results.html ↔ import.html 상호 링크.

**R6. "한눈에"의 정의 (C vs D vs E)**

index.html / server.js 코드는 더 이상 손대지 않습니다.

### 📂 편집 순서 (공통)
1. `D:\설문조사html\public\questions.json` 파일을 **메모장** 또는 **VS Code**로 연다
2. 아래 규칙대로 내용을 고친다
3. **저장**한다 (Ctrl+S)
4. **서버를 재시작한다** — 기존 검은 창을 닫고, 새 터미널에서:
   ```
   cd D:\설문조사html
   node server.js
   ```
   (문구만 바꿨고 변수는 안 바꿨다면 서버 재시작 없이 브라우저 F5만으로도 반영됨. 변수·필수 변경은 반드시 재시작)
5. 브라우저에서 `http://localhost:3000/index.html` 새로고침(F5)

> ⚠️ JSON 문법 주의: 각 항목은 쉼표(,)로 구분, 마지막 항목 뒤에는 쉼표 없음. 큰따옴표(")만 사용. 자신 없으면 기존 줄을 **복사**해서 값만 바꾸는 게 가장 안전.

### 🔑 문항 1개의 구조
```json
{ "var": "score_spicy", "header": "매운맛", "label": "매운맛은 어떠셨나요?", "type": "rating", "required": true }
```
| 키 | 뜻 | 비고 |
|---|---|---|
| `var` | 영문 변수명 (영문+밑줄, 고유) | 엑셀 저장·집계의 키. 중복 금지 |
| `header` | 엑셀에 들어갈 한글 열 제목 | |
| `label` | 화면에 보이는 질문 문구 | |
| `type` | 문항 유형 (아래 표) | |
| `required` | 필수 여부 | `true` / `false` |
| `hint` | (선택) 질문 아래 회색 설명 | |
| `placeholder` | (선택) 입력칸 흐린 예시 글자 | text/textarea |
| `options` | (선택) 보기 목록 | select/segment |
| `ends` | (선택) 양끝 라벨 | scale은 3개, nps는 2개 |
| `emojis`,`labels` | (선택) 평점 이모지·라벨 5개씩 | rating |

### 🎚️ 문항 유형(type) 8종 — 복사용 템플릿
```json
{ "var":"q_text",   "header":"이름",   "label":"이름",       "type":"text",     "required":true, "placeholder":"홍길동" }
{ "var":"q_email",  "header":"이메일", "label":"이메일",     "type":"email",    "required":false, "placeholder":"a@b.com" }
{ "var":"q_long",   "header":"의견",   "label":"자유 의견",  "type":"textarea", "required":false, "placeholder":"자유롭게" }
{ "var":"q_sel",    "header":"연령",   "label":"연령대",     "type":"select",   "required":true, "options":["10대","20대","30대"] }
{ "var":"q_seg",    "header":"성별",   "label":"성별",       "type":"segment",  "required":true, "options":["여성","남성"] }
{ "var":"q_rate",   "header":"맛",     "label":"맛 평가",    "type":"rating",   "required":true }
{ "var":"q_scale",  "header":"단맛",   "label":"단맛 강도",  "type":"scale",    "required":true, "ends":["부족","적당","과함"] }
{ "var":"q_nps",    "header":"추천",   "label":"추천 의향",  "type":"nps",      "required":true, "ends":["아니다","그렇다"] }
{ "var":"q_agree",  "header":"동의",   "label":"활용에 동의합니다", "type":"consent", "required":false }
```
- **text/email** = 한 줄 입력 · **textarea** = 여러 줄 · **select** = 드롭다운 · **segment** = 가로 버튼
- **rating** = 1~5 이모지 점수 · **scale** = 1~5 강도(부족↔과함, 틸색) · **nps** = 0~10 · **consent** = 동의 체크박스

### ✅ 자주 하는 작업 따라하기

**(1) 질문 문구만 바꾸기** → 해당 문항의 `"label"` 값만 교체 → 저장 → F5

**(2) 선택지 추가** → 해당 `select`/`segment` 문항의 `"options"` 배열에 항목 추가:
```json
"options": ["10대", "20대", "30대", "40대", "50대 이상", "60대 이상"]
```

**(3) 새 질문 추가** ← 가장 중요
1. 원하는 섹션의 `"questions": [ ... ]` 안에서 비슷한 문항 한 줄을 **복사**
2. 바로 위 줄 끝에 쉼표(,)가 있는지 확인
3. 복사한 줄의 `var`(고유한 새 영문명), `header`, `label`을 수정
   ```json
   { "var": "score_spicy", "header": "매운맛", "label": "매운맛은 적당했나요?", "type": "rating", "required": true }
   ```
4. 저장 → **서버 재시작**(`node server.js`) → 끝. 엑셀에 '매운맛' 열이 자동 추가됨

**(4) 질문 삭제** → 해당 문항 줄(`{ ... }`)을 통째로 삭제 → 앞 줄의 쉼표 정리 → 저장 → 재시작

**(5) 섹션(단계) 추가** → `"sections"` 배열에 통째로 추가:
```json
{
  "title": "포장·배송 평가",
  "questions": [
    { "var":"score_package", "header":"포장", "label":"포장 상태", "type":"rating", "required":true }
  ]
}
```

**(6) 필수 ↔ 선택 전환** → 해당 문항의 `"required"` 값을 `true`/`false`로 변경

**(7) 제품명·안내 문구·완료 메시지** → 맨 위 `"meta"` 블록의 `title`/`subtitle`/`doneText` 등 수정

### ⚠️ 주의
- `var`는 절대 중복되면 안 되고, 영문+숫자+밑줄(`_`)만 사용 (한글·공백·하이픈 금지)
- `var`를 새로 추가/변경하면 **서버 재시작 필수** (서버가 시작될 때 questions.json을 읽기 때문)
- `results.html` 종합 대시보드는 **기본 관능 구조(맛·향·식감·단맛 등)** 기준으로 명칭이 짜여 있음. 변수명을 크게 바꾸면 그 항목은 0으로 표시될 수 있음 → 그럴 땐 어떤 구조든 자동 분석하는 **`import.html`** 을 쓰거나, results.html 수정을 요청.

# 🌐 5차 확장 — 공개 배포(WiFi 없이 어디서나) (2026-07-01)

## (1) QR이 폰에서 안 되던 문제 해결
- 원인: QR에 `localhost` 주소가 담겨 폰에선 접근 불가(`null 접근 불가`).
- 해결: `server.js`에 `GET /api/host`(LAN IP 반환) 추가, `results.html` QR이 **PC 내부 IP(예 172.30.1.52)** 자동 사용 + localhost면 경고.
- 단, LAN 방식은 **참여자가 PC와 같은 WiFi**일 때만 가능.

## (2) WiFi 없는 환경(각자 LTE) → 공개 배포
- 선택: **A. 클라우드 배포**, 단 **새 계정 없이 + 응답 영구보관** 요구 → **Google Apps Script 웹앱** 채택.
- 구글이 설문 화면을 직접 호스팅, 응답은 **구글시트에 영구 저장**. 서버·GitHub·Render·요금 불필요(기존 구글 계정만).
- 자료: `assets/apps-script/Code.gs`(CONFIG에 questions.json 주입) + `Form.html`(google.script.run로 제출),
  가이드 `references/public-deploy.md`.
- 결과 보기: 구글시트 실시간 / .xlsx 내려받아 `import.html`로 대시보드화.
- 예시 번들 생성: `D:\신메뉴투표설문\apps-script\` (Code.gs+Form.html) — 바로 배포 가능.

## (3) 접속 방식 정리
| 상황 | 방식 |
|---|---|
| 매장·부스(같은 WiFi) | 로컬 Node + QR(LAN IP) — 지금 그대로 |
| 어디서나·LTE·행사 | Apps Script 웹앱 공개 배포 (구글시트 저장) |
| 대규모·완전 무난 | 구글폼 + `import.html` 정리 |
